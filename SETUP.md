# Інструкція з налаштування

## Вимоги

- Node.js 18+ 
- PostgreSQL 12+ (або SQLite для тестування)
- npm або yarn

## Крок 1: Встановлення залежностей

```bash
npm run install:all
```

## Крок 2: Налаштування бази даних

### Варіант 1: PostgreSQL (рекомендовано)

1. Створіть базу даних:
```sql
CREATE DATABASE kiosk_db;
```

2. Створіть файл `backend/.env`:
```env
PORT=3001
JWT_SECRET=your-secret-key-change-in-production
DATABASE_URL=postgresql://user:password@localhost:5432/kiosk_db
NODE_ENV=development
```

### Варіант 2: SQLite (для швидкого тестування)

Можна змінити `backend/src/db/init.ts` для використання SQLite замість PostgreSQL.

## Крок 3: Запуск

### Запуск всього проєкту (backend + frontend):

```bash
npm run dev
```

### Або окремо:

```bash
# Backend
npm run dev:backend

# Frontend (в іншому терміналі)
npm run dev:frontend
```

## Доступ

- Frontend: http://localhost:5173
- Backend API: http://localhost:3001

## Тестові дані для входу

- **Логін**: `admin`
- **Пароль**: `admin123`
- **Роль**: Адміністратор

## Структура бази даних

База даних створюється автоматично при першому запуску backend. Створюються такі таблиці:

- `kiosks` - ларьки
- `users` - користувачі (адміністратори та продавці)
- `products` - товари
- `sales` - продажі
- `schedule` - графік роботи

## Функціонал

### Адміністратор:
- ✅ Дашборд зі статистикою
- ✅ Управління товарами
- ✅ Перегляд продажів
- ✅ Управління ларьками
- ✅ Управління продавцями
- ✅ Графік роботи

### Продавець:
- ✅ Панель продавця
- ✅ Перегляд товарів свого ларька
- ✅ Продаж товарів
- ✅ Перегляд власних продажів
- ✅ Перегляд своєї комісії (12%)

## Примітки

- Комісія продавця розраховується автоматично: 12% від суми продажу
- При продажі товару кількість автоматично зменшується
- Статус товару змінюється на "Немає в наявності" коли кількість = 0
